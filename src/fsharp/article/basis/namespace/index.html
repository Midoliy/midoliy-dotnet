名前空間

プログラムが大きくなってくると、ファイルやデータを種類ごとにフォルダ分けするように、モジュールや型を種類ごとに分けて管理したくなってきます。
そういったときに、名前空間(= namespace) が役に立ちます。

基本的に名前空間もフォルダ管理と同じように入れ子で管理することが多いです。
例えば、以下のようなフォルダ構成を考えてみます。



ファイルを管理する際には、大項目フォルダの中に関連する中項目フォルダを作成し、さらにその中に小項目フォルダやファイルを作成することで管理をしやすくします。
上記のような構成にすることによって、"何が" "どこに" あるのかがわかりやすくなるため、目的のものを探すのが容易になります。
プログラムにおいては、このフォルダの部分が 名前空間 の部分にあたります。

これと同じ考え方で、プログラムを作成する場合にも関連する項目ごとに名前空間を作成します。
FSharp.Coreの名前空間を参考に見てみましょう。



上記の図は、Fsharp.Coreの名前空間をフォルダ構成に似せたものとなります。
これを見ると、ArrayモジュールやListモジュールなどのコレクションに関連するものはMicrosoft.FSharp.Collectionsの中に定義されていることがわかります。
また、OptionモジュールやPrintfモジュール、Stringモジュールなどの中核を成す機能についてはMicrosoft.FSharp.Coreの中に定義されていることがわかります。

このように関連する項目ごとにグルーピングするための機能が名前空間なわけです。
また、フォルダ管理をする際にはフォルダ階層を「プロジェクト/ドキュメント」のようにスラッシュやバックスラッシュで区切りますが、名前空間では「Microsoft.FSharp.Collections」のように ドット で区切ります。



名前空間の使い方

名前空間を利用するには namespace というキーワードを利用します。
また、名前空間はファイルの最上位要素として宣言しなければなりません。

つまり、namespaceを使う前にmoduleやtypeなどを宣言することができないというわけです。
それでは今回も FSharp.Core の名前空間をサンプルに、コードを見ていきましょう。
(*** 構文 ***)
namespace 名前空間名
namespace Microsoft
namespace FSharp
namespace Core

module Sample =
    [<EntryPoint>]
    let main argv =
        printfn "Hello world!!"
        0
Microsoft.FSharp.Core名前空間を定義したい場合には、上記のように順番通りnamespaceを定義していきます。
ただ、一般的には以下のように「ドット」で繋いで一行で定義します。
namespace Microsoft.FSharp.Core

module Sample =
    [<EntryPoint>]
    let main argv =
        printfn "Hello world!!"
        0

namespaceを複数行にすることで入れ子にできることは上記の例でわかったと思います。
この性質を利用することで以下のようなことができます。
namespace Microsoft.FSharp.Core
// この Sample モジュールは正確には
//      Microsoft.FSharp.Core.Sample
// となっている
module Sample =
    let f x = 
        x * x


namespace Sub   // この行から Microsoft.FSharp.Core.Sub 名前空間となる
// この Sample モジュールは正確には
//      Microsoft.FSharp.Core.Sub.Sample
//                            ^^^
// となっている
module Sample =
    [<EntryPoint>]
    let main argv =
        printfn "Hello world!!"
        0
このように同一ファイル内で入れ子になっている名前空間を利用することが可能となっています。
ただし最初の名前空間は、最初のmoduleやtypeの宣言よりも前に定義されていなければなりません。
このルールがあるため、以下のコードはエラーとなります。
// ファイル内に namespace がある場合、
// 最初の namespace定義 よりも前に module や type の宣言はできない
module Sample =
    let f x = 
        x * x

namespace Microsoft.FSharp.Core

module Sample =
    let f x = 
        x * x

namespace Sub
module Sample =
    []
    let main argv =
        printfn "Hello world!!"
        0

名前空間はアプリケーション名や組織名などで始まることが多いですが、特に決まったルールはありません。
しかし、あまり細かく名前空間を分けすぎると利用者側も作成者側もわかりにくくなってしまうため、自分が適切だと思う粒度感で名前空間を定義するようにしましょう。
